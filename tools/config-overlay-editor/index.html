OMP config overlay editor

Create host configuration JSON and config overlay JSON. These objects stay separate from global module definitions and artifact binaries.

Host configuration

Use this as the input for generating host-specific overlays.

This object is intentionally opaque to OMP core. Generator scripts can decide which values they need.

Config overlay

Use this when a host needs configuration files or SQL that must not be part of the global artifact/module object.

For zip packages you may use source/path. Standalone JSON should use inlineSql.

Output

The generated JSON can be imported through Portal, the HostAgent import folder, or an installer package library.
